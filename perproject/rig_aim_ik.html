IK spline & AIM Constraints



Requirements

Create a Spline Setup for the Chain Links.
Add only AIM constraints for the Cardan Joint

Challenges

The curve in the IK spline setup has to be connected to the locators, so they can be manipulated outside the system.
How to use only the aim contraint to make the Cardan Joint work?

How I tackled the problem : IK Spline

For the IK spline System, I used clustors to create points with Transform. Then a Point contraint was used with locators to achieve the user controlled system you currently see.

Final Solution

The aim constraint has two requirements. The Aim Vector and the Up Vector. Aligning the Aim Vector is the easiest problem. Aligning the up vector has three variation in a maya scene. It can point to the scene up (y axis) or the object up (the up vector of the obj) or it can align with object rotation. Typically this is handled using a second locator. But for this problem, we were given only one locator.
The base of the joint works only in one direction, the z direction. This was the easiest to solve. But the top of the joint works in two directions. For a long time, I used only the x and z-axis for this which did not work right. The reason being, the x and z where the axis of the lcoator and the direction the locator moves in, not the joint. The joint moves in y and z. Once I fixed this directional problem, my joint worked in sync. Voila!
