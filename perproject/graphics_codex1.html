Graphics - Transforms Intro Project

Project Specifications

Create Cube Primitives that can be translated, rotated and scaled to produce the desired output.
Three different scenes are created, the CornelL box for scaling cubes, the spiral stairs to rotate and automate the creation process, and finally the complex model scene, where only cubes are used to create a model of complexity. In all of these scenes only a cube is created.

Project was implemented in two platforms: Unity 3D and G3D (opensource)
Project guide and specification coutesy - Morgan McGuire's The Graphics Codex : Cubes



Scene 1

Seven scaled cubes were used to create this scene. The Cornell Box is a real-world box at Cornell University that has been long used for photorealistic rendering experiments.The idea is that by constructing a real scene containing only well-measured geometric primitives.



Hiccups

The Cornell Box was a fairly straigt forward project. I had to fine tune the translation and scale for the cubes that make up the walls. I also spent sometime makeing sure the cubes inside where spaced and rotated just so.


Scene 2

The staircase in a spiral fashion was a harder endeavor than the Cornell box. This involved translation the next stair correctly and then rotating it. Simple rotating a long cube might look like a spiral staircase from an angle, but it doesnâ€™t really create one.

This took a while to understand and figure out the correlation between the stairs created. I ended up using the translation with respect to the previous stair. This worked in the end.


Hiccups

The stiars were very straigt forward to solve in Unity. I used the previous stair as the parent, an rotated the next stair with respect to that. This created the nice curvature I was looking for. The implementation of this was not staright forward at all in G3D. In this engine, I couldn't create the parent / child relationship I had in Unity and hence had to work with the limitations I has. I ended up creating a simple staircase in this instance.


Scene 3

The final scene by far was the hardest to figure out in terms of idea. I finally ended up creating a Lego scene. The entire scene is created with cubes. They are each individual entities (girl, bench and dog). Each entity was then individually scaled, rotated and translated to match the scene.


Hiccups

The most tricky part of this project was coming up with a complex enough idea that will look not only visually pleasing but also complex enough to challenge me. I picked a scene from a lefo set (thich is essentially made of trnasformed cubes) and created by project. I created lego pieces and build this digitally very similar to the actual building process.


Conclusion

Of the three, Cornell Box was not an issue, I understood what was needed and executed them with out issue. Because I understood Unity better, I decided to implement the same projects in Unity as well. This was to help me understand the concepts in a platform I was comfortable with and to something I can relate with faster. The interesting scene and the stair case where also simple tasks in Unity. I translated the stairs by using the previous stair as the parent. This also meant I could curve the stair to any angle I wished. On the other hand, G3D proved to be a fickle friend. I tried really hard to bring this effect in G3D and failed miserably. I was able to create a single stair, but automating it meant knowing more about the backend code and learning where to place my function. I looked through all the code created as sample scenes and found something that could be implemented for my automation. Although the rotation happened with respect to the origin and not the previous stair iin G3D unlike Unity. This is still something I still haven't yet managed to solve to my satisfaction. To ensure the solution I have is not incorrect, I referred to the other project images and realized this was the only way in which the stairs were created for the G3D Engine in question and the project in play.
